## 프로그래머 확인 항목

### 1. 사운드 매니저 — BGM 재생 시 예외 가능성
- **내용**: `SoundManager.cs`에서 효과음(`PlaySFX`)은 오디오 소스가 없을 때를 대비한 확인 코드가 있는데, 배경음(`PlayBGMForScene`/`PlayBGMForBoss`) 쪽에는 같은 확인이 빠져 있습니다. 오디오 소스 컴포넌트가 어떤 이유로든 없는 상태에서 씬 BGM이 재생되려 하면 예외가 날 수 있습니다.
- **재현 조건**: 아직 재현된 적 없음 — 현재 정상 작동 중이라면 오디오 소스가 잘 붙어 있다는 뜻이라 급하지 않습니다.
- **관련 파일**: `Assets/Scripts/SoundManager.cs`
- **결정 필요 여부**: 아니오 — Editor에서 오디오 소스 부착 여부만 확인되면 프로그래머가 방어 코드 한 줄 추가로 마무리 가능.

### 2. CraftUI 안에 남아있는 미사용 코드
- **내용**: `CraftUI.cs`에 주석 처리된 채 방치된 필드 2개와 메서드 1개가 있습니다. 실제로 쓰이지 않아서 삭제해도 동작에 영향이 없습니다.
- **관련 파일**: `Assets/Scripts/UI/CraftUI.cs` (16, 29, 34, 85~97번째 줄 부근)
- **결정 필요 여부**: 아니오 — 삭제만 하면 되는 정리 작업. CraftUI/InventoryUI 자체의 큰 구조 변경은 별도로 보류 중이니 이 부분과 섞지 않을 예정입니다.

### 3. 코루틴이 컴포넌트 비활성화 시 자동으로 멈추지 않음
- **내용**: 방 전환 담당(`RoomManager`), 화면 UI 담당(`UIManager`), 플레이어(`Player`) 쪽 일부 진행 중 동작(코루틴)이 해당 오브젝트가 꺼질 때 자동으로 멈추도록 되어 있지 않습니다. 현재 씬 전환 흐름에서는 문제가 된 적이 없고, 나중에 "일시적으로 껐다 켜는" 기능이 추가될 때 신경 써야 하는 부분입니다.
- **재현 조건**: 없음 — 예방 차원의 참고 사항.
- **관련 파일**: `RoomManager.cs`, `UIManager.cs`, `Player.cs`
- **결정 필요 여부**: 지금은 아니오. 구조를 바꾸는 작업이라 필요해지는 시점에 별도로 설계하는 게 안전합니다.

### 4. 인벤토리 포션 스택 판정 (참고만)
- **내용**: 제작된 포션이 인벤토리에서 같은 칸으로 합쳐질지 판단하는 로직이 이름/수치 비교 방식을 씁니다. 이론적으로는 서로 다른 포션이 우연히 같은 칸에 합쳐질 가능성이 있지만, 실제로 관찰된 적은 없습니다.
- **관련 파일**: `Assets/Scripts/Inventory.cs`
- **결정 필요 여부**: 아니오 — 인벤토리에서 이상한 스택 현상을 발견하면 이 부분부터 확인해주세요.

## Unity Editor 작업자 확인 항목

### 1. SoundManager 프리팹 — 오디오 소스 부착 확인
- **확인 방법**: SoundManager가 붙은 오브젝트(프리팹 또는 씬 오브젝트)를 열어 `Audio Source` 컴포넌트가 있는지 확인.
- **결정 필요 여부**: 없음 — 확인만 해주시면 됩니다. 없다면 알려주세요(프로그래머 확인 항목 1번과 연결).

### 2. CraftUI 벌로우즈(풀무) 애니메이션 스프라이트 확인
- **확인 방법**: `Resources` 폴더에 `bellow1`, `bellow2`, `bellow3`, `bellow4` 스프라이트 4장이 전부 있는지 확인. 하나라도 없으면 애니메이션이 정지 이미지로 대체되어 보입니다(에러는 안 남).
- **결정 필요 여부**: 없음 — 있으면 넘어가도 됩니다.

### 3. PlayerBeamHitGuard 부착 여부 확인
- **확인 방법**: 퀸 보스 관련 프리팹에 `PlayerBeamHitGuard` 컴포넌트가 붙어 있는지 확인. 코드상 실제로 사용되는 곳이 없어서, 붙어 있지 않다면 다음 정리 라운드에서 파일 자체를 삭제할 예정입니다.
- **결정 필요 여부**: 예 — 삭제해도 되는지 최종 확인 필요(부착돼 있다면 삭제 보류).
