# 플레이테스트 가이드 — QA 콘솔 (2026-07-06)

에디터에서 플레이를 시작하면 QA 콘솔이 자동 활성됩니다 (콘솔에 `[QA] BugTestConsole 활성` 로그).
`UNITY_EDITOR || DEVELOPMENT_BUILD` 전용 — 릴리즈 빌드에는 포함되지 않습니다.
파일: `Assets/Scripts/ForDebugging_HG/BugTestConsole.cs`

## 단축키

| 키 | 기능 | 용도 |
|---|---|---|
| **F1** | 현재 방 4방향 연결 + 보스방 목록(좌표·경로 유무) 콘솔 출력 | 방 연결 데이터 확인 |
| **F2** | 다음 보스방으로 자동 이동 (누를 때마다 보스방 순환) | 보스전 테스트 동선 단축 |
| **F3** | 활성 보스에 maxHP 25% 데미지 | 페이즈 전환 테스트 (즉사 아님) |
| **F4** | 콜라이더 시각화 토글 — 빨강=정밀 히트박스, 청록=플레이어 본체, 노랑=보스 투사체 | 판정 확인. **켜 두면 투사체에 노란 박스가 보이니 시각 확인 시 꺼 둘 것** |
| **F5** | 활성 보스 즉시 처치 | 보스전 종료/문 재개방 테스트 |
| **F6~F9** | 방 이동 북/남/서/동 | 이동 동선 단축 |

- F2/F6~F9는 `RoomManager.RequestMove` 공식 경로만 사용 — 보스 잠금/쿨다운이면 내부에서 거부되고, 막힌 통로면 전환이 롤백됩니다. 임의 좌표 텔레포트 기능은 의도적으로 없습니다(방 상태와 어긋나는 갈라짐 방지).
- F3/F5는 `BossHealth.TakeDamage` 공식 경로 — 페이즈 핸들러/사망/문 개방 체인이 그대로 동작합니다. 보스가 무적 상태면 건드리지 않고 경고만 출력합니다.

## 자동 감시 (키 없음 — 경고 없으면 통과)

1초 간격으로 검사하고, 문제 발생 시 상태 변화당 1회만 경고 로그를 남깁니다.

| 감시 | 의미 |
|---|---|
| 몬스터 방 셀 이탈 | BUG-3/4 회귀 의심 — 몬스터가 방 밖으로 나감 |
| 보스전 중 열린 문 | BUG-2 회귀 의심 — 보스전인데 문이 트리거 상태 |
| 플레이어 비정상 위치 | 전환 갈라짐/벽 바깥 갇힘 — 플레이어가 카메라(현재 방) 셀과 다르거나 지면 타일 밖 |

## 회귀 테스트 체크리스트 (요약)

1. 아무 방에서나 사망 → Restart → 맵/방 정상 로드, 패널 가림 없음.
2. 보스전 시작 → 4방향 문 차단 + "You cannot flee!" → F5 처치 → 문 재개방.
3. 문/벽 근처에서 몬스터 접촉·포션 넉백 → 방 밖으로 나가지 않음.
4. F4 켜고 보스 투사체를 스치게 → 노랑 박스가 빨강 박스에 닿을 때만 명중.
5. 막힌 방향으로 F6~F9 → "전환 롤백" 경고와 함께 제자리 유지.
6. 플레이 종료 시 "Some objects were not cleaned up" 경고 없음.
