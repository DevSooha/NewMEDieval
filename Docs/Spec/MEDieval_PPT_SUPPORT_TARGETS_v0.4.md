# MEDieval PPT 기획서 기반 보조 목표 명세 v0.4

> 용도: Fable5 구현 감사에서 `MEDieval_Fable5_감사용_목표명세_정리본_v0.2.md`와 별도로 투입하는 보조 기준 문서.  
> 성격: PPT 텍스트 사본에서 추출한 프로젝트 전체 기능 의도, UI/시스템 플로우, 구형 보스전 정리.  
> 변경: `메디벌_시스템기획_설정의 사본.txt`는 사용자 지시에 따라 제외. PotionUI의 RectMask2D/Mask 구조는 현행 구현 기준이 아니므로 감사 필수 기준에서 제외. 크래프팅/물약 입력과 수량 처리는 사용자 확정 기준을 반영.

- 피그마/사진에만 존재하는 좌표, 크기, 위치: 텍스트 감사에서 절대값 판정하지 않는다.

## 1. 조작키 공통 기준

### 1-1. 원문상 기본 조작

| 입력 | 기능 |
|---|---|
| 방향키 / WASD | 캐릭터 이동, 대사 선택지 이동 |
| Z | 상호작용, 대사 진행, 선택지 확정, 공격, 폭탄 설치 |
| 마우스 우클릭 | 필드 공격/폭탄 설치 가능. 크래프팅 재료 선택 및 완성 물약 장착 진입에도 사용 |
| X | 취소, 뒤로가기 |
| C | 무기창 스왑. 누를 때마다 무기창 원이 화살표 방향으로 한 칸 이동 |
| 마우스 | 크래프팅 씬에서는 마우스 조작 중심. 좌클릭 문구보다 우클릭 기준을 우선 |

### 1-2. 감사 포인트

- 대화/UI 중 Z 입력이 공격으로 새지 않는지 확인한다.
- 크래프팅 씬에서는 필드 이동/공격 입력이 막히는지 확인한다.
- C 입력으로 무기 슬롯 순환이 되는지 확인한다.
- 필드 공격은 Z와 우클릭 모두 가능한지 확인한다.
- 크래프팅/물약 조작은 좌클릭 문구보다 우클릭 기준을 우선한다.

## 2. 세이브/로드 시스템

### 2-1. 목적

- 모닥불과 상호작용하여 현재 진행 상태를 저장한다.
- 타이틀의 Continue로 마지막 저장 데이터를 로드한다.
- QA 및 진행상황 유지, 엔딩 분기 기록 기반으로 사용한다.

### 2-2. 세이브 UI 흐름

1. 맵 내 모닥불과 상호작용한다.
2. `SAVE`와 `POTION` 선택지가 있는 UI가 뜬다.
3. `POTION` 선택 시 0.5초 후 크래프팅 화면으로 이동한다.
4. `SAVE` 선택 시 0.1초 후 `YES/NO` 선택지가 뜬다.
5. `YES` 선택 시 즉시 저장하고 `SAVE`가 `SAVED!`로 바뀐다.
6. `NO` 선택 시 저장 UI가 닫히거나 선택창이 사라진다.

### 2-3. 저장 단위

- 단일 세이브만 지원한다.
- 다중 세이브 슬롯은 요구사항이 아니다.

### 2-4. 저장 대상

필수/우선 저장 대상:

- 현재 플레이 중인 맵 ID
- 마지막으로 저장한 모닥불 ID
- 현재 체력
- 보유 아이템 ID
- 보유 수량
- 무기 장착 아이템 정보

후순위/나중 구현 대상:

- 주요 이벤트/엔딩 분기 상태
- 메인퀘스트 아이템 보유 여부
- 각 보스 처치 여부
- 보스 클리어 이후 영구 맵 변화
- 특정 보스전 진입 해금 여부
- 엔딩 분기에 따라 고정되는 월드/NPC 상태
- 세이브 데이터 버전 관리

### 2-5. 저장 제외 권장 대상

- 일반 적의 일시 상태
- 전투 중 일시 피격 상태
- 맵의 일시 드랍템
- 일회성 이펙트
- 플레이어가 던진 폭탄 등 런타임 임시 오브젝트

### 2-6. 로드 기준

- 저장 파일이 있으면 타이틀 `CONTINUE`에서 마지막 저장 데이터를 로드한다.
- 저장 파일이 없으면 `CONTINUE`는 비활성화되거나 눌러도 동작하지 않는다.
- 로드 후 플레이어는 마지막 저장 모닥불 위치 기준으로 복귀한다.

### 2-7. 권장 로드 순서

1. 맵 로드
2. 맵 영구 변화 반영
3. 플레이어 상태 반영
4. 인벤토리 반영
5. UI 반영

### 2-8. 감사 포인트

- 저장 중 중복 입력을 막는지 확인한다.
- 저장/로드 후 인벤토리 ID, 수량, 장착 슬롯이 깨지지 않는지 확인한다.
- 저장 파일이 없을 때 Continue가 안전하게 막히는지 확인한다.
- 보류 항목을 필수 실패로 잡지 않는다.

## 3. 크래프팅 시스템

### 3-1. 크래프팅 씬 진입

1. 모닥불과 상호작용한다.
2. 선택지 중 `물약 제조`를 선택한다.
3. 0.5초 페이드아웃/페이드인 후 크래프팅 씬으로 이동한다.

### 3-2. 인벤토리 슬롯

| 구분 | 기준 |
|---|---|
| 재료 인벤 | 물약 제조에 사용할 재료 보관 |
| 완성 물약 인벤 | 제조된 물약 보관 |
| 재료 페이지 | 6개 슬롯 단위 |
| 완성 물약 페이지 | 5개 슬롯 단위 |
| 페이지 이동 | 다음 페이지 아이템이 없으면 1페이지로 순환 |
| 페이지 전환 시간 | 0.05초 |
| 툴팁 | 슬롯 hover 시 즉시 아이템 이름 표시 |
| 재료 스택 | 한 슬롯당 최대 99개 |
| 완성 물약 스택 | 현행 구현 기준 우선. 사용자 추정 기준은 99. 구현이 99면 정상, 다르면 실제 구현값을 기록 |

### 3-3. 재료 선택

- 재료는 총 2개 선택해야 한다.
- 같은 아이템 2개 선택 가능.
- 재료 2개 선택 전에는 산소주입기/미니게임이 작동하지 않아야 한다.
- 재료 선택 시 솥 물 튐 애니메이션과 효과음이 재생된다.
- 재료 선택 입력은 우클릭 기준이다.
- 재료는 선택/제조 중 임시 차감될 수 있으나, 실패/리셋/나가기/취소 시 복구되어야 한다.
- 제작 성공 시에만 최종적으로 재료 2개가 소모된다.

### 3-4. Bellow MiniGame

#### 시작 조건

- 재료 2개 선택 후 산소주입기를 조작하면 온도바가 뜬다.
- 레버를 일정 거리 이상 아래로 드래그하면 미니게임이 시작된다.
- 크래프팅 씬에서는 마우스만 사용한다.

#### 조작/수치

- 총 게임 시간: 6초
- 드래그 중 게이지 상승: +28px/sec
- 드래그하지 않을 때 자연 하락: -10px/sec
- 산소주입기 레버 복귀 애니메이션: 0.25초
- 온도 조절바 길이: 128px
- 게이지 범위: 0~100

#### 온도 판정

| 결과 | px 범위 | Gauge 범위 | 최종 판정 |
|---|---:|---:|---|
| 고온 | 0~31px | 75~100 | gauge >= 75 |
| 중온 | 32~63px | 50~75 | gauge >= 50 |
| 저온 | 64~95px | 25~50 | gauge >= 25 |
| 실패 | 96~127px | 0~25 | else |

### 3-5. 완성 처리

- 6초 타이머 종료 시 온도 결과를 확정한다.
- 실패면 물약 제조가 실패하고 초기화된다.
- 성공 시 `PotionData`를 생성한다.
- 이름 규칙: `온도 + 1번 아이템 앞 이름 + 2번 아이템 뒤 이름 + 물약`.
- 완성 시 `item_get` 사운드가 재생된다.
- 완성 물약은 물약 인벤에 추가된다.
- 완성 물약 스택 최대는 현행 구현 기준을 따른다. 사용자 추정 기준은 99다.
- PotionUI의 구체 표시 방식은 현행 구현 기준을 따른다. RectMask2D/Mask 미사용을 실패로 잡지 않는다.

### 3-6. 무기창 장착

- 완성 물약 인벤에서 원하는 물약을 우클릭하면 무기 인벤 4칸을 고르는 상태가 된다.
- 선택 중에는 해당 물약/무기 인벤 외 화면이 dimmed 70% 처리된다.
- 무기 인벤 4칸 중 하나를 선택하면 해당 칸에 물약이 배정된다.
- 장착 시 해당 물약은 1개 소비/이동되는 기준으로 감사한다.
- 이미 물약이 배치된 무기 칸을 선택하면 기존 물약은 물약 인벤으로 돌아가고 새 물약이 배정된다.
- 선택한 물약을 다시 누르거나 취소 입력을 하면 장착 선택이 취소된다.

### 3-7. 리셋/나가기

#### 리셋

- 리셋 버튼 클릭 시 즉시 “제조하고 있는 물약을 폐기하시겠습니까?” 선택창이 뜬다.
- Yes: 진행 중인 제조 상태 삭제, 크래프팅 초기 상태 복귀.
- Yes: 진행 중 선택 재료는 복구된다. 최종 소모로 확정하지 않는다.
- No: 선택창만 닫는다.

#### 나가기

- 나가기 버튼 클릭 시 “나가시겠습니까? 변경사항은 저장되지 않습니다” 선택창이 뜬다.
- Yes: 0.5초 페이드아웃/페이드인으로 모닥불 스팟 복귀.
- No: 선택창만 닫는다.
- 진행 중 선택 재료는 복구된다. 완성되지 않은 PotionData/PotionUI/임시 상태는 폐기된다.

### 3-8. 감사 포인트

- 재료 2개 전 미니게임 시작 차단.
- 게이지 상승/하락/최종 판정 수치.
- 실패 시 데이터/인벤/UI 상태 정리.
- 리셋/나가기 확인창과 재료 복구 처리.
- 장착 슬롯 교체/취소 처리.
- 완성 물약 1개 장착 소비와 스택 수량 보존.

## 4. 물약 표시/PotionData 보조 기준

### 4-1. 감사 기준으로 남길 것

- 물약은 두 재료와 온도 결과를 기반으로 생성된다.
- 첫 번째 재료와 두 번째 재료의 순서가 결과 이름/효과/패턴에 영향을 줄 수 있다.
- 실패 시 PotionData/PotionUI가 생성되지 않거나 최종 인벤에 남지 않아야 하며, 선택 재료는 복구되어야 한다.
- 완성 물약은 인벤토리에서 이름을 확인할 수 있어야 한다.
- 필드 전환 후 장착 상태가 유지되어야 한다.

### 4-2. 감사 기준에서 제외할 것

- `RectMask2D`, `TopArea`, `BottomArea`, `TopImage`, `BottomImage`, `BottleFrame` 조합을 필수 구현으로 보지 않는다.
- 별도 결과 스프라이트 없이 UI Mask로 합성해야 한다는 요구를 현행 필수 기준으로 보지 않는다.
- 현재 구현이 다른 방식이면 코드 기준으로 실제 표시/데이터/장착 동작만 감사한다.

## 5. 보스전 공통 의도

### 5-1. 차별화 포인트

- 땅따먹기/장애물 제거 여부가 회피 공간과 탄막 차단에 영향을 준다.
- 플레이어의 공격이 플레이어 자신에게도 영향을 줄 수 있다.
- 메인 속성 외에 서브 속성으로 빛/어둠/독 등을 활용한다.

### 5-2. 공통 규칙

- 플레이어는 보스에게 닿아도 데미지를 입지 않는다.
- 보스 공격 1대는 플레이어 체력 1을 깎는다.
- 보스 체력은 화면에 표시되지 않는다.
- 보스/플레이어의 속성 기술은 일부 오브젝트 파괴/상태 변화에 영향을 준다.

## 6. 세 마녀 보스전

### 6-1. 기본 정보

- 보스명: 세 마녀
- 등장 맵: 봄 맵 1_5
- 컨셉: 불/물/전기를 다루는 예언가형 마녀
- 전투 형태: 원거리 3패턴, 근거리 1패턴
- 목표 플레이: 장애물 제거로 회피 공간 확보, 속성 상성 맞추기
- 참고 HP: 14400
- 목표 플레이 시간: 2분

### 6-2. 페이즈 의도

- 1페이즈: HP 60%까지, 마녀 3명이 물/불/전기 공격을 천천히 번갈아 사용
- 2페이즈: HP 40%까지, 교체 빈도와 원거리 공격 속도 증가
- 3페이즈: HP 10% 미만, 원거리 공격을 멈추고 근거리 돌진

주의: 60%~40%, 40%~10% 구간 처리와 확률 의미가 불명확하므로 정밀 감사 기준으로 쓰지 않는다.
