# MEDieval Fable5 감사용 목표 명세 정리본 v0.2

목적: 원문 기획 문서를 Fable5/AI 감사가 “구현 목표 기준”으로 사용할 수 있도록 정리한 버전이다.  
원문에서 수치·순서가 명확한 항목은 **확정 기준**으로 두고, 원문이 깨졌거나 모호한 항목은 **감사 보류/TODO**로 분리했다.

---

## 0. 감사 공통 기준

### 0.1 감사 관점
구현 감사는 아래 순서로 확인한다.

1. 시작 조건/트리거
2. 상태 전이/페이즈 전환
3. 공격 패턴 순서
4. 타이밍/대기/인터벌
5. 좌표/방향/속도
6. 충돌 판정/피격 가능 시점
7. 데미지/회복/상태이상
8. 오브젝트 생성/유지/소멸
9. 사망/승리/전투 리셋/씬 전환
10. 코루틴·타임라인 중복 실행, 잔여 탄막, 이벤트 구독 누수

### 0.2 아트 미구현 처리
원문에 “애니메이션/아트 미구현”이라고 적힌 항목은 감사 기준에서 **애니메이션 존재 여부가 아니라 해당 시간 인터벌과 판정 발생 여부**만 확인한다.

### 0.3 좌표/칸 기준
- 1칸 = 32px.
- `1칸 옆`은 대상 위치 기준 ±32px.
- 속도 표기는 원문 단위와 px/s를 함께 유지한다.
- 프리팹 배치 좌표가 에디터 수동 배치 전제인 경우, 코드 감사에서는 “참조 연결/배열/리스트/자식 오브젝트 순회”만 확인하고 정확한 씬 배치는 별도 수동 검증으로 분리한다.

### 0.4 플레이어 피격 기본값
기획서에 “플레이어 데미지 없음” 또는 “체력 회복”으로 명시된 탄막/효과를 제외하면, 모든 탄막/장판/오브젝트는 플레이어에게 데미지 1을 적용하는 것을 기본 기준으로 둔다.

# 1. Nemi BossCombat 감사용 목표 명세

## 1.1 메타 정보

| 항목 | 기준 |
|---|---|
| 보스명 | Nemi |
| 등장 방 | `spr_1 BOSS` |
| 총 HP | 5200 |
| 약점 속성 | 없음. 모든 속성 공격을 계수 없이 받음 |
| 접촉 피해 | 플레이어가 보스에게 닿아도 데미지 없음 |
| 보스 공격 피해 | 공격 1회 적중 시 플레이어 체력 -1 |
| 보스 HP UI | 화면에 표시하지 않음 |

## 1.2 전투 시작/종료

### 전투 시작
1. 전투 승낙 시 화면을 0.5초 동안 흰색 페이드아웃한다.
2. 0.5초 동안 원래 화면으로 돌아온다.
3. 페이드 중 플레이어 위치를 `playerStartPositionTF = (0.5, -7.9)`로 고정한다.
4. 전투 시작 즉시 페이즈 1의 `Hand of Time`을 시작한다.

### 승리
1. 네미 HP가 0 이하가 되면 플레이어 승리.
2. 엔딩 씬으로 전환한다.
3. 해피엔딩/카르마엔딩 조건은 현재 단일 변수로 제어한다.

### 플레이어 사망
1. 승리 전에 플레이어가 사망하면 화면을 0.5초 동안 검은색 페이드아웃한다.
2. 0.5초 동안 원래 화면으로 돌아온다.
3. 플레이어를 `playerStartPositionTF`로 이동시킨다.
4. 진행 중이던 패턴/탄막/코루틴은 전투 재시작 전 정리되어야 한다.

## 1.3 페이즈 상태

| 상태 | HP 조건 | 실행 |
|---|---:|---|
| Phase 1 | HP 2200 이상 ~ 5200 이하 | `Hand of Time` → `Fate Sever` 반복 |
| Phase 2 진입 | HP 2200 미만 최초 진입 | 초기 위치로 이동 후 `Pearl Beam` 1회 |
| Phase 2 반복 | Pearl Beam 이후 HP 0 초과 | `Enhanced Hand of Time` → `Fate Sever` 반복 |

### 감사 보류/TODO
- 전투 시작 시 보스 위치 `(0, 0.5)`와 Phase 2 진입 위치 `(0.5, 0)`이 다르다. 의도/오타 확인 필요.
- HP 2200 미만 시 현재 패턴을 즉시 중단하는지, 현재 패턴/딜레이 종료 후 전환하는지 원문 기준이 부족하다. 최소 감사 기준은 “Pearl Beam 1회만 실행, 이후 Phase 1 루프 재개 금지”로 둔다.

## 1.4 Phase 1: Hand of Time

### 순서
1. 보스 공격 시전 인터벌 0.5초.
2. 시전 직후 Hand of Time 탄막 방향/생성 위치 결정.
3. 0.5초 동안 탄막 페이드인.
4. 탄막 발사.
5. 탄막은 화면 이탈 전까지 유지된다.
6. 플레이어에게 닿아도 탄막은 사라지지 않는다.

### 탄막 종류

| 종류 | 크기 | 생성 위치 | 이동 방향 | 속도 |
|---|---:|---|---|---:|
| 짧은 축 | 64x64 | 화면 위쪽 | 아래 | 5칸/s = 160px/s |
| 긴 축 | 32x96 | 화면 오른쪽 | 왼쪽 | 5칸/s = 160px/s |

### 생성 후보 범위
- 스크린 해상도: 1024x576
- 좌우 UI 가림 영역: 각 64px
- 실제 맵 노출 후보 범위: 896x576
- 짧은 축 후보: 64px 단위, 총 14칸
- 긴 축 후보: 32px 단위, 총 18칸
- 짧은 축은 플레이어 x좌표가 속한 격자의 화면 위쪽에 생성한다.
- 긴 축은 플레이어 y좌표가 속한 격자의 화면 오른쪽에 생성한다.

### 충돌/피해
- 탄막 생성 위치에 플레이어가 서 있으면 피해를 받을 수 있다.
- 발사 중 플레이어에게 닿으면 플레이어 체력 -1.
- 탄막은 플레이어에게 닿아도 제거하지 않는다.

## 1.5 Phase 1/2 공통: Fate Sever

### 순서
1. Hand of Time 시전 직후, 보스는 탄막 진행과 별개로 현재 위치에서 2초 휴식한다.
2. 0.5초 동안 Fate Sever 시전 인터벌.
3. 0.5초 대기.
4. 플레이어 왼쪽 1칸 옆으로 순간이동.
5. 0.1초 대기.
6. 플레이어 오른쪽 1칸 옆으로 순간이동.
7. 0.1초 대기.
8. 플레이어 왼쪽 1칸 옆으로 순간이동.
9. 0.5초 공격 인터벌.
10. 공격 판정 시점에 플레이어가 보스 오른쪽 1칸 옆에 있으면 플레이어 체력 -1.
11. 그 자리에서 3초 대기.
12. 현재 페이즈의 다음 루프를 시작한다.

### 특이사항
- Nemi는 공중에 떠 있는 설정이다.
- 순간이동 위치가 Field 타일이 아니어도 해당 위치에 존재할 수 있어야 한다.

## 1.6 Phase 2 최초 1회: Pearl Beam

### 순서
1. HP가 2200 미만으로 처음 내려가면 네미가 초기 위치로 이동한다.
2. Pearl Beam을 정확히 1회 실행한다.
3. 홀수 위치 1,3,5,7,9에서 0.8초 동안 마법진 생성.
4. 0.3초 만에 빔이 화면 최상단까지 발사된다.
5. 빔이 0.5초 유지된다.
6. 홀수 빔이 꺼진 직후 짝수 위치 2,4,6,8,10에서 같은 순서 반복.
7. Pearl Beam 이후 보스는 제자리에서 3초 대기한다.
8. 이후 Phase 2 반복 패턴으로 넘어간다.

### 판정
- Pearl Beam은 하단 마법진에서 북쪽으로 진행한다.
- 특정 x좌표에 마법진이 생기면 해당 x좌표부터 x+64px까지의 세로 열 전체가 데미지 영역이다.
- 데미지 영역에 플레이어가 닿으면 체력 -1.

## 1.7 Phase 2 반복: Enhanced Hand of Time

### Phase 1과 다른 점
1. 0.5초 시전 인터벌.
2. 기존 짧은 축/긴 축 탄막 외에 Bedimmed Wall 4개를 추가 생성한다.
3. 짧은 축/긴 축 탄막은 6칸/s로 이동한다.
4. Bedimmed Wall은 4칸/s로 이동한다.
5. 모든 탄막은 화면 이탈 전까지 유지된다.
6. Bedimmed Wall과 보스 순간이동 위치가 겹쳐도 보스는 순간이동 가능해야 한다.

### Bedimmed Wall 배치

| 번호 | 좌표 | 방향 |
|---:|---|---|
| 1 | (-11.5, 6.5) | 오른쪽 |
| 2 | (11.5, 2.5) | 왼쪽 |
| 3 | (-11.5, -2.5) | 오른쪽 |
| 4 | (11.5, -6.5) | 왼쪽 |

### 감사 보류/TODO
- 원문에 “0.5/0.7초간 탄막 생성”이라고 되어 있어, 짧은/긴 축과 Bedimmed Wall의 페이드인 시간이 각각 0.5초/0.7초인지 확인 필요.

# 2. Hidden BossCombat 감사용 목표 명세

## 2.1 메타 정보

| 항목 | 기준 |
|---|---|
| 등장 조건 | 해피엔딩을 본 후 2회차 플레이 |
| 등장 방 | `win_0` |
| 전투 시작 트리거 | `FinalBossBattleTrigger` 2개 중 하나를 플레이어가 밟음 |
| 추정 총 HP | 5400 |
| Phase 1 | HP 2500 이상 |
| Phase 2 | HP 2500 미만 |
| 접촉/공격 피해 | 보스 공격 1회 적중 시 플레이어 체력 -1로 해석 |

### 감사 보류/TODO
- 총 HP는 원문 첫 줄의 Phase 1 범위 `2500~5400`에서 추정한 값이다. 별도 HP 선언이 있으면 그 값을 우선한다.

## 2.2 전투 시작/종료

### 전투 시작
1. 해피엔딩 이후 2회차에서 플레이어가 `win_0`에 들어오면 보스는 `startPositionTF`에 서 있다.
2. 플레이어가 `FinalBossBattleTrigger`를 밟으면 전투가 시작된다.
3. 전투 시작 즉시 `Latent Thorn`을 시작한다.

### 승리
1. 보스 HP가 0이 되는 순간 모든 공격을 멈춘다.
2. 남아 있는 탄막/공격 오브젝트를 제거 또는 비활성화한다.
3. 히든 씬을 로딩한다.

### 플레이어 사망
1. 화면을 0.5초 동안 검은색 페이드아웃한다.
2. 플레이어를 `win_0` 입장 계단 좌표로 이동시킨다.
3. 보스전 시작 전 상태로 되돌린다.
4. 진행 중이던 공격/탄막/코루틴은 모두 정리되어야 한다.

## 2.3 Phase 1 루프

### 구조
`Latent Thorn`과 `Carma Excision`이 겹쳐 진행되는 루프다.

1. `Latent Thorn` 시작.
2. Thorn 타임라인이 진행되는 동안 `Carma Excision`도 실행.
3. `Carma Excision` 종료 후 3초 딜레이.
4. 다시 `Latent Thorn`부터 반복.

### 감사 보류/TODO
- Carma Excision 1번에 “플레이어는 그 자리에서 2초간 딜레이”라고 되어 있으나, 패턴 문맥상 “보스는 그 자리에서 2초간 딜레이”일 가능성이 높다. 구현 감사 시 원문 오타로 보고 보스 딜레이로 해석하는 것을 권장한다.

## 2.4 Phase 1: Latent Thorn

### 순서
1. 보스가 `startPositionTF`에서 시작한다.
2. 0.5초 동안 검을 앞으로 휘두르는 시전 인터벌을 가진다.
3. 시전 중 플레이어가 보스 앞 1칸, 즉 32x32px 영역에 있으면 플레이어 체력 -1.
4. 0.5초 시전 후 Thorn 타임라인 시작.
5. 0.5초 동안 바닥 마법진 생성.
6. 0.1초 동안 가시 상승.
7. 가시가 5초 유지.
8. 가시가 0.3초 동안 소멸.
9. 가시는 상승 시작부터 소멸 완료 전까지 총 5.4초 동안 데미지 판정을 가진다.

### 충돌/피해
- 플레이어가 활성 가시에 닿으면 체력 -1.
- 가시는 플레이어에게 닿아도 Destroy되지 않는다.

## 2.5 Phase 1/2 공통: Carma Excision

### 순서
1. Latent Thorn 소환 모션 이후 보스가 2초 대기한다.
2. 플레이어 왼쪽 1칸 옆으로 순간이동 후 0.5초 대기.
3. 플레이어 오른쪽 1칸 옆으로 순간이동 후 0.1초 대기.
4. 플레이어 왼쪽 1칸 옆으로 순간이동 후 0.1초 대기.
5. 왼쪽 자리에서 0.4초 동안 페이크 공격 인터벌.
6. 플레이어 오른쪽 1칸 옆으로 순간이동 후 0.1초 대기.
7. 0.3초 동안 진짜 공격 인터벌.
8. 공격 후 그 자리에서 3초 딜레이.
9. 이후 현재 페이즈 루프로 복귀한다.

### 특이사항
- Tilemap이 깔려 있지 않은 곳, 즉 Ground가 아닌 곳에도 보스는 순간이동 가능해야 한다.

### 감사 보류/TODO
- 진짜 공격의 데미지 범위가 “보스 위치의 오른쪽 한 칸 위”라고 되어 있으나 원문에 변경 가능성이 표시되어 있다. 이 판정은 구현 감사 전에 확정 필요.
- 페이크 공격 0.4초는 기존 0.5초 공격의 중간 캔슬이다. 실제 데미지가 들어가지 않아야 한다.

## 2.6 Phase 2 최초 1회: Bedimmed Wall 응용 필살기

### 발동 조건
1. HP가 2500 미만으로 내려가면 Phase 2 진입.
2. 사용 중인 패턴과 딜레이가 끝난 뒤 `startPositionTF`로 이동한다.
3. 필살기는 Phase 2 진입 시 정확히 1회만 사용한다.

### 순서
1. 0.5초 동안 검을 휘두르는 시전 인터벌.
2. 시전 중 플레이어가 보스 앞 1칸, 즉 32x32px 영역에 있으면 플레이어 체력 -1.
3. 시전 이후 미리 배치된 어둠벽 8개가 0.7초 동안 나타난다.
4. 시전 이후 `Diamond Flickering`도 발동한다.
5. 벽은 각각 상/하/좌/우로 6칸/s = 192px/s 속도로 발사된다.
6. 벽 발사 이후 보스는 2초 동안 제자리 딜레이.
7. 어둠벽은 화면 이탈 전까지 유지되고, 플레이어에게 닿아도 소멸하지 않는다.

### 감사 보류/TODO
- 8개 어둠벽의 정확한 좌표와 방향은 “맵 예시를 보고 에디터에서 직접 배치”라고 되어 있어 코드 감사 기준만으로는 확인 불가하다.
- `Diamond Flickering`의 효과/타이밍/판정 명세가 없다. 현재 문서만으로는 엄격 감사 불가하다.

## 2.7 Phase 2 반복: Hidden Hand of Time 응용

### 순서
1. 필살기와 필살기 딜레이 이후, 또는 반복 루프의 Carma Excision 딜레이 이후 보스가 `startPositionTF`로 이동한다.
2. 0.5초 대기.
3. 0.5초 시전 인터벌.

## 4.3 기사 혼령 패턴

9. 파편 크기는 2칸 x 2칸 = 64x64px.
10. 플레이어가 파편에 닿으면 플레이어 체력 -1.

## 6.4 서브 속성 시스템

### 개요
- 탄막은 메인 속성 외에 서브 속성 `Light`, `Dark`, `Poison`을 가질 수 있다.
- 서브 속성은 탄막이 대상에게 닿는 순간 적용된다.
- 서브 속성 공격은 플레이어 → 보스/적 방향에만 성립한다.
- 적이 플레이어에게 서브 속성 효과를 적용하는 경우는 없다.

### 배율

| 서브 속성 | 효과 | 비고 |
|---|---|---|
| Light | 어둠 속성 대상에게 데미지 x2 | 기존 데미지 합산 후 2배 |
| Dark | 빛 속성 대상에게 데미지 x2 | 기존 데미지 합산 후 2배 |
| Poison | 배율 없음 | 필드 효과/DoT 규칙으로 처리 |

## 6.5 Poison / MORBIDMIRE High DoT

### 고온 효과 정의
- 적에게 닿을 시 10초 동안 2초마다 현재 체력의 5% 감소.
- 총 5회 tick.
- Poison 서브 속성 부여.

### 계산 기준
- 매 tick 후 현재 체력은 `현재 체력 x 0.95`.
- 5tick 후 잔여 체력은 약 `0.95^5 = 0.7738`.
- 10초 후 누적 피해는 약 22~23%.

### Poison 필드 효과
- 물 속성 필드 또는 물 속성 오브젝트에 Poison 탄막이 닿으면 Poison 탄막은 즉시 소멸한다.
- 예: 물웅덩이, 물 속성 보스 공격.
- Poison 필드 효과는 우선순위 낮음. 시간 부족 시 생략 가능.

## 6.6 HALLOWBLAZE 저온/중온 회전 속도 변화

### 기준
- 패턴 3 Tornado에만 적용한다.
- 적용 시점은 탄막 발사 후부터 적/플레이어에게 닿기 직전까지다.
- 저온은 시계방향 회전 속도 감소.
- 중온은 시계방향 회전 속도 증가.

### 감사 보류/TODO
- `물약목록.md` 현재 테스트 목록에는 HALLOWBLAZE High만 명시되어 있다. 저온/중온 회전 속도 효과가 현재 구현 범위인지 확인 필요.
- “탄막이 적/플레이어에게 닿으면 탄막은 소멸하지만, 시계방향 이동은 변경된 속도 그대로 유지”라는 문장은 표현상 모순이 있다. 소멸 후 이동 유지가 아니라, 소멸 전까지 속도 변경 유지로 해석하는 것을 권장한다.

## 6.7 넉백 시스템

### 공통 원칙
- 넉백 방향은 피격 방향의 반대 방향.
- 플레이어 넉백 거리: 32px = 1칸.
- 일반 적 넉백 거리: 64px = 2칸.
- 필드에서 적과 플레이어가 직접 충돌하면 양측 모두 넉백 발생.

### 감사 보류/TODO

# 8. Fable5 감사 지시문 예시

아래 문장을 감사 프롬프트 뒤에 붙이면 된다.

```text
아래 목표 명세를 기준으로 현재 구현을 감사해라.
원문 기획 설명이 아니라 “감사용 목표 명세”를 기준으로 판정한다.

감사 방식:
1. 구현 완료/부분 구현/미구현/기획 불명확/회귀 위험으로 분류.
2. 각 항목마다 관련 파일, 메서드, 코루틴, 프리팹/씬 연결 필요 여부를 적는다.
3. 수치가 명시된 항목은 HP, 시간, 속도, 좌표, 데미지, 지속시간을 실제 코드 상수와 비교한다.
4. 코루틴 중복 실행, 페이즈 전환 중 잔여 탄막, 사망/승리 시 cleanup 누락, 이벤트 구독 해제 누락을 우선 확인한다.
5. 원문이 모호한 항목은 임의로 구현 정답을 만들지 말고 “기획 확정 필요”로 분리한다.
6. 아트 미구현 애니메이션은 애니메이션 존재 여부가 아니라 해당 시간 인터벌과 판정 발생 여부만 검사한다.
7. `물약목록.md`와 `재료-온도별 효과 확인해줘.txt`가 충돌하면 `물약목록.md`를 현재 감사 기준으로 우선한다.
8. `물약 폭탄 전체.txt`와 `물약폭탄2번패턴로직.txt`가 패턴2에서 충돌하면 `물약폭탄2번패턴로직.txt`를 우선한다.
9. 최종 출력은 다음 형식으로 한다.

[요약]
- 전체 상태:
- 치명 이슈:
- 기획 확정 필요:
- 구현 감사 우선순위:

[상세]
ID / 기능 / 기대 동작 / 실제 구현 / 판정 / 근거 파일·라인 / 수정 제안
```
